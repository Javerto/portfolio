Veri Temizliği: Neden Analiz Süresinin %80'ini Alır?

Yayınlanma Tarihi: 24 Nisan 2026
%80

Araştırmalar gösteriyor ki: Veri bilimcilerin zamanının %80'i veri temizleme ve hazırlama süreçlerinde geçiyor.

Giriş: Veri Temizliğinin Kritik Önemi

Veri bilimi genellikle karmaşık algoritmalar ve gelişmiş makine öğrenmesi modelleriyle ilişkilendirilir. Ancak, veri biliminin en zaman alıcı yönü model geliştirmek değil, veriyi hazırlamaktır. "Garbage in, garbage out" prensibi burada devreye giriyor: Kirli veriyle çalışmak, ne kadar gelişmiş model kullanırsanız kullanın, güvenilmez sonuçlar üretir.

Forrester Araştırması (2023):
  • Şirketlerin %25'inden fazlası, kötü veri kalitesi nedeniyle yılda 5 milyon dolardan fazla kayıp yaşıyor
  • Veri temizleme süreçleri otomatikleştirilmediğinde bu maliyetler katlanıyor

Veri Temizliği Nedir?

Veri temizliği, ham verideki hataları, tutarsızlıkları ve kalite sorunlarını belirleyip düzelterek analiz için uygun hale getirme sürecidir. Bu süreç şunları içerir:

  • Eksik değerlerin yönetimi: Boş alanlar, null değerler
  • Tekrar eden kayıtların temizlenmesi: Yinelenen müşteri veya işlem kayıtları
  • Yapısal hataların düzeltilmesi: Format tutarsızlıkları, yazım hataları
  • Aykırı değerlerin tespiti: İş mantığına uymayan uç değerler
  • Veri tipi standardizasyonu: Tutarlı veri tipleri ve formatlar

Veri Temizliğinin Analiz Süresindeki %80 Payı: Gerçek mi?

Bu istatistik ilk bakışta abartılı görünebilir, ancak pratikte oldukça gerçekçidir. İşte nedenleri:

1. Veri Kaynaklarının Çeşitliliği

Modern işletmeler veriyi onlarca farklı kaynaktan toplar:

  • CRM sistemleri
  • ERP yazılımları
  • Web analitik araçları
  • Sosyal medya API'ları
  • IoT sensörleri
  • Harici veri sağlayıcılar

Her kaynağın kendi veri yapısı, formatı ve kalite standartları vardır.

2. Veri Kalitesi Sorunlarının Çeşitliliği

En Yaygın Veri Kalitesi Sorunları:
  • Eksik değerler (%15-30 oranında)
  • Tekrar eden kayıtlar (%5-15)
  • Format tutarsızlıkları (%10-20)
  • Aykırı değerler (%3-10)
  • Yapısal hatalar (%5-15)

Veri Temizliğinin Temel Teknikleri

1. Format Standardizasyonu

Tutarsız formatlar veriyi sıralama, filtreleme ve birleştirme işlemlerini zorlaştırır.

# Python'da tarih formatı standardizasyonu
import pandas as pd
from datetime import datetime

# Farklı formatlarda tarihler
dates = ['01/02/26', '2026-02-01', 'Feb 1, 2026', '01.02.2026']

# Standardize et
standard_dates = []
for date in dates:
    try:
        # Farklı formatları deneyerek standart formata dönüştür
        parsed_date = pd.to_datetime(date, infer_datetime_format=True)
        standard_dates.append(parsed_date.strftime('%Y-%m-%d'))
    except:
        standard_dates.append('Geçersiz tarih')

print(standard_dates)  # ['2026-01-02', '2026-02-01', '2026-02-01', '2026-01-02']

2. Tekrar Eden Kayıtların Temizlenmesi

Tekrar eden kayıtlar metrikleri şişirir ve analizleri çarpıtır.

# Pandas ile tekrar eden kayıtları temizleme
import pandas as pd

# Örnek veri seti
data = {
    'customer_id': [101, 102, 101, 103, 102],
    'name': ['Ali', 'Ayşe', 'Ali', 'Mehmet', 'Ayşe'],
    'purchase_amount': [150, 200, 150, 300, 200]
}

df = pd.DataFrame(data)

# Tekrar eden kayıtları kaldır
cleaned_df = df.drop_duplicates(subset=['customer_id'], keep='last')
print(f"Önceki kayıt sayısı: {len(df)}")
print(f"Sonraki kayıt sayısı: {len(cleaned_df)}")

3. Eksik Değerlerin Yönetimi

Eksik değerler için tek bir "doğru" yaklaşım yoktur. Bağlama göre değişir:

# Eksik değer stratejileri
import pandas as pd
import numpy as np

# Örnek veri
sales_data = pd.DataFrame({
    'month': ['Jan', 'Feb', 'Mar', 'Apr', 'May'],
    'revenue': [1000, np.nan, 1200, np.nan, 1500],
    'customers': [50, 55, np.nan, 60, 65]
})

# Strateji 1: Ortalama ile doldurma
sales_data['revenue'].fillna(sales_data['revenue'].mean(), inplace=True)

# Strateji 2: Önceki değerle doldurma
sales_data['customers'].fillna(method='ffill', inplace=True)

# Strateji 3: Satırı tamamen kaldırma
sales_data_cleaned = sales_data.dropna()

Adım 1: Veri Profili Oluşturma

Değişiklik yapmadan önce veriyi anlamak için profilleme yapın:

# Veri profilleme
import pandas as pd

def data_profile(df):
    profile = {
        'total_rows': len(df),
        'total_columns': len(df.columns),
        'missing_values': df.isnull().sum().sum(),
        'missing_percentage': (df.isnull().sum().sum() / (len(df) * len(df.columns))) * 100,
        'duplicate_rows': df.duplicated().sum(),
        'data_types': df.dtypes.to_dict()
    }
    return profile

# Profili görüntüle
profile = data_profile(df)
for key, value in profile.items():
    print(f"{key}: {value}")

Adım 5: Aykırı Değerleri Tespit Etme ve Yönetme

# Aykırı değer tespiti
import numpy as np

def detect_outliers_iqr(data):
    Q1 = np.percentile(data, 25)
    Q3 = np.percentile(data, 75)
    IQR = Q3 - Q1
    lower_bound = Q1 - 1.5 * IQR
    upper_bound = Q3 + 1.5 * IQR
    
    outliers = [x for x in data if x < lower_bound or x > upper_bound]
    return outliers

# Örnek kullanım
revenue_data = [1000, 1200, 1100, 15000, 1050, 1150]  # 15000 aykırı değer
outliers = detect_outliers_iqr(revenue_data)
print(f"Aykırı değerler: {outliers}")
